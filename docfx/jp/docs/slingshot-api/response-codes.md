# 200 (Success)

API リクエストを送信すると、ステータス コードを含む  [HTTPS](https://developer.mozilla.org/ja/docs/Glossary/HTTPS) (Hypertext Transfer Protocol Secure) レスポンスが返されます。

API へのリクエストが成功しない場合があります。このような場合、失敗に関する情報 (エラー メッセージとエラー コード) を HTTP 応答のステータス コードで確認できます。応答本文は [JSON](https://www.json.org/json-ja.html) 形式です。

これらのエラーの理由はさまざまです。以下に、プロセスで表示されるさまざまなタイプのステータス コードを示します。

| コード | 説明|
-------------------------------------------------------------------- | ------------------ | ------------------ | ------------------ |
|200 (Success)|送信されたリクエストは正常に実行され、データを使用できるようになりました。 |
| 201 (Created) |リクエストは正常に実行され、リソースが作成されました。 |
|204 (No Content)|リクエストは成功しました。現在使用しているページから移動する必要はありません。|
| 400 (Bad Request) |パラメーターが欠落しているか、形式が正しくないため、要求は処理されませんでした。何が問題だったのかについての情報を含むエラー配列が返されます。|
| 403 (Forbidden) |サーバーは要求を理解していますが、要求を承認できません。これは、たとえば、アクセスせずにオブジェクトを読み込もうとしたときに発生する可能性があります。再認証の必要はありません。  |
| 404 (Not Found) |T要求されたリソースがサーバーで見つかりません。これは、たとえば、指定されたオブジェクトが存在しないことが原因である可能性があります。 |
|429 (Too Many Requests)|一定時間内にリクエストが多すぎます。Slingshot の API レート制限の詳細については、[こちら](rate-limits.md)をご覧ください。|
|500 (Internal Server Error)| Slingshot の側で予期しないことが発生しました。|
