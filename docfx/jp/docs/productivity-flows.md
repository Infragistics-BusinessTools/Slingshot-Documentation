# 生産性フロー

生産性は常に企業や個人にとって最優先事項でしたが、最近ではこれまで以上に注目を集めています。
在宅勤務は、絶え間ないアプリの切り替えをなくす Slingshot のようなオールインワン アプリの必要性を高めます。それだけでも役立ちますが、エンゲージメント、効率を促進し、何よりも時間を節約するには、生産性の流れも重要です。  

## 生産性フローとは?
これは、さまざまな機能間のシームレスな統合を利用して、より多くの仕事を遂行するのに役立つ Slingshot を使用する方法です。これは、効率を高め、Slingshot のエクスペリエンスを向上させるための優れた方法です。

## シナリオのリスト

以下に、生産性を高めることができるシナリオのリストを示します:

| カテゴリ | 生産性フロー | シナリオ例 | 使い方|
|---|---|---|---|
| タスク >> ディスカッション | タスクからディスカッションを開始します。 | タスクを確認しているときに、優先度が低いとラベル付けされている、チーム全体にとって重要なタスクを発見します。すぐにチームと話し合う必要があることにすぐに気付きます。| タスクのオーバーフロー メニューに移動し、[ディスカッションの開始] を選択し、新しいディスカッションを開始するか、既存のディスカッションを使用するワークスペースまたはプロジェクトに移動します。 |
| タスク >> チャット | タスクからチャットを開始します。 | タスクを更新しているときに、別の Slingshot ユーザーが割り当てられていることを思い出します。進捗状況についての会話が必要なので、プライベート メッセージを介してタスクの共同作業者に連絡します。 | タスクのオーバーフロー メニューに移動し、[プライベート チャットの開始] を選択 > 既存のチャットを使用するか、ユーザー (またはグループ) との新しいチャットを開始します。 |
| ディスカッション >> タスク | ディスカッションからタスクを作成します。 | 白熱した議論の末、チームメートとの共通点を見つけたとします。最終的なメッセージで、行動計画を決定しました。これらのメッセージを直接使用して、ディスカッション トピックを離れることなくタスクを作成する必要があります。 | メッセージのオーバーフロー メニューに移動し、[タスクの作成] を選択し、新しいタスクを作成するワークスペースまたはプロジェクトを選択します。 |
| ピン固定 >> チャット | ピン固定されたファイルからチャットを開始します。 | チーム ボードにピン固定されたスプレッドシートで作業していて、それを完了するにはチームメートの助けが必要です。プライベート チャットでクイック アップデートに行くことにしたとします。 | ファイルのオーバーフロー メニューに移動し、[プライベート チャットの開始] を選択 > 既存のチャットを使用するか、新しいチャットを開始 > ファイルが最初にリストに追加された場合は、ファイルのアクセス許可を設定します。 |
| ピン固定 >> タスク | ピン固定されたファイルからタスクを作成します。 |プロジェクトにピン固定されているコンテンツを確認しているときに、すぐに注意が必要なファイルを見つけました。ただし、レビューはまだ完了していないため、[ピン固定] タブを離れずにタスクを作成する必要があります。 | ファイルのオーバーフロー メニューに移動し、[タスクの作成] を選択し、新しいタスクを作成するワークスペースまたはプロジェクトを選択します。 |
| ピン固定 >> ディスカッション | ピン固定されたファイルからディスカッションを開始します。 | プロジェクトのコンテンツにあるファイルに出くわし、すぐにその目的について話し合う必要があると判断しました。チームの [ピン固定] タブを離れたり、生産性の流れを中断したりすることなく、これを行うことができます。 | ファイルのオーバーフロー メニューに移動し、[ディスカッションを開始] を選択し、ディスカッションを作成するワークスペースまたはプロジェクトを選択します。 |
| 分析 >> ピン固定 | ダッシュボードをワークスペースまたはプロジェクトにピン固定します。 | チームが定期的に追跡する必要がある KPI があります。これらのダッシュボードをワークスペースまたはプロジェクトにピン固定して、誰もがすぐにアクセスできるようにします。 | ダッシュボードのオーバーフロー メニューに移動し、[ピン固定に追加] > ピン固定を追加するワークスペースまたはプロジェクトを選択します。 |
| 分析 >> チャット | ダッシュボードからチャットを開始します。 | ダッシュボードで追跡された KPI の偏差について話し合いたいとします。この目的のために、チャット メッセージでダッシュボードを別の Slingshot ユーザーまたはグループと共有できます。さらに、KPI を表示する視覚化のスクリーンショットを含めることができます。これらすべてを [分析] タブを離れることなく実行できます。| ダッシュボードのオーバーフロー メニューに移動 > [プライベート チャットの開始] > 既存のチャット / グループ チャットを選択するか、新しいチャットを開始します。 |
| 分析 >> タスク | ダッシュボードからタスクを作成します。 | [分析] タブのダッシュボードの情報を解析している間、すでにいくつかのタスクを作成し、見つけた問題を解決することを望んでいます。必要なタスクを作成し、後でダッシュボード分析を続行するために、[分析] タブを離れる必要はありません。| ダッシュボードのオーバーフロー メニューに移動し、[タスクの作成] を選択し、新しいタスクを作成するワークスペースまたはプロジェクトを選択します。 |
| 分析 >> ディスカッション | ダッシュボードからディスカッションを開始します。 |  [分析] でダッシュボードを解析していると、サポート ケースの増加についての認識を高める必要があることがわかるとします。全員の注意を引くために、ディスカッションを開始することにしました。 | ダッシュボードのオーバーフロー メニューに移動し、[ディスカッションを開始] を選択し、ディスカッションを作成するワークスペースまたはプロジェクトを選択します。 |
| チャット >> タスク | チャット メッセージからタスクを作成します。 | チームメイトとチャットすると、問題の解決策が一緒に見つかったとします。ここで、会話の終了後にアクションを実行するタスクをすばやく追加する必要があります。チャットを終了せずにこれを行うことができます。 | チャット メッセージのオーバーフロー メニューに移動し、[タスクの作成] を選択し、タスクを作成するワークスペースまたはプロジェクトを選択します。 |
| チャットの添付ファイル >> |チャット メッセージの添付ファイルからブックマークおよびタスクの作成、またはディスカッションの開始を行うことができます。 | チャット メッセージで送信された添付ファイルは、チャットを離れることなく生産性フローを開始するためにすべての参加者が使用できます。各添付ファイルを個別に使用して、ブックマークに追加したり、タスクを作成したり、ディスカッションを開始したりできます。 | 添付ファイルのオーバーフロー メニューに移動し、[ブックマーク]、[タスクの作成]、または [ディスカッションを開始] を選択して、 望ましい場所に移動します。 |
| タスクの添付ファイル >> |ブックマーク、ピン固定に追加、タスクの作成、またはタスクの添付ファイルからのディスカッションを開始することができます。| タスク内のすべての添付ファイルは、タスクとは関係なく、生産性フローを開始するために使用できます。添付ファイル付きのピン固定を追加したり、ブックマークを追加したり、タスクを作成したり、ディスカッションを開始したりできます。| 添付ファイルのオーバーフロー メニューに移動し、[ブックマーク]、[ピン固定に追加]、[タスクの作成]、[ディスカッションを開始] のいずれかを選択し、目的の場所に移動します。|
| ブックマーク >> | ピン固定に追加したり、チャットを開始したり、タスクを作成したり、ブックマークからディスカッションを開始したりできます。 | メッセージ、タスク、ファイル、添付ファイルなど、Slingshot のすべてをブックマークできます。また、任意のブックマークを使用して、チャット、ディスカッションの開始、タスクの作成、またはピン固定の追加を行うことができます。| ブックマークのオーバーフロー メニューに移動 > [ピン固定に追加] / [プライベート チャットの開始] / [タスクの作成] / [ディスカッションを開始] >目的の場所に移動します。|
